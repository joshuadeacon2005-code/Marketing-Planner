# USER CREATION DEBUG & FIX PROMPT

## Context
Your Marketing Planner application is throwing "Failed to create user. Please try again." errors. This document provides a systematic debugging approach.

## System Architecture
- **Framework**: Express.js + React + TypeScript
- **Database**: PostgreSQL with Drizzle ORM
- **Auth**: JWT with bcrypt password hashing
- **Deployment**: Replit environment

```typescript
      return res.status(400).json({ message: "Invalid input", errors: error.errors });

    // Handle database constraint violations
    const errorMessage = error instanceof Error ? error.message : String(error);
    if (errorMessage.includes('duplicate') || errorMessage.includes('unique')) {
      return res.status(400).json({ message: "A user with this email already exists" });

    res.status(500).json({
```

### 7. COMMON ISSUES & SOLUTIONS

#### Issue 1: Database Connection Pool Exhausted
**Symptoms:** Timeout errors, "too many clients"
**Solution:**
```typescript
// In server/db.ts, add connection pool config:
export const pool = new Pool({ 
  connectionString: process.env.DATABASE_URL,
  max: 20,  // maximum pool size
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});
```

#### Issue 2: Password Hashing Fails
**Symptoms:** Error during bcrypt.hash()
**Solution:**
```bash
# Reinstall bcryptjs
npm uninstall bcryptjs
npm install bcryptjs@2.4.3
```

#### Issue 3: Database Constraint Violation
**Symptoms:** "unique constraint violation"
**Solution:**
- Check for duplicate email in database
- Ensure email is properly lowercase-normalized
```typescript
// In storage.ts, normalize email:
async createUser(user: InsertUser): Promise<User> {
  const normalizedUser = {
    ...user,
    email: user.email.toLowerCase().trim()
  };
  const [created] = await db.insert(users).values(normalizedUser).returning();
  return created;
}
```

#### Issue 4: Missing SESSION_SECRET
**Symptoms:** "SESSION_SECRET environment variable is required"
**Solution:**
```bash
# Add to Replit Secrets
SESSION_SECRET=your-super-secret-jwt-key-here-minimum-32-characters
```

#### Issue 5: Foreign Key Constraint (regionId)
**Symptoms:** "foreign key constraint violation"
**Solution:**
```sql
-- Check if the regionId exists:
SELECT * FROM regions WHERE id = 'the-region-id-from-request';

-- If no regions exist, create one:
INSERT INTO regions (name, code, timezone) 
VALUES ('Global', 'GLOBAL', 'UTC');
```

#### Issue 6: Missing Required Environment Variables
**Check all required env vars:**
```bash
echo "DATABASE_URL: ${DATABASE_URL:+SET}"
echo "SESSION_SECRET: ${SESSION_SECRET:+SET}"
echo "SLACK_BOT_TOKEN: ${SLACK_BOT_TOKEN:+SET}"
```

### 3. Implement Retry Logic

```typescript
async function createUserWithRetry(userData, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await storage.createUser(userData);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)));
    }
  }
}
```
